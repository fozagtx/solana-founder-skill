## When to Use

Use for:

- Case studies
- Pilot summaries
- Grant milestone updates
- Launch proof posts
- Pull quotes
- Objection maps
- Win/loss analysis
- Messaging shifts from user feedback

## Evidence Rules

- Cite every specific claim when source anchors exist.
- If citations are missing, label claims as uncited.
- Do not invent metrics, quotes, user names, revenue, usage, partners, or integrations.
- Preserve user language.
- List gaps to close.

## Output Standards

For proof assets, produce:

- before state
- trigger
- intervention
- outcome
- proof bullets
- quotes if available
- gaps to close
- next ask

For win/loss, count patterns and separate stated reason from likely real reason.
