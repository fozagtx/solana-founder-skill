## When to Use

Use for:

- Choosing one first acquisition channel
- Writing outreach messages
- Planning developer dry-runs
- Finding first integrations
- Building partner asks
- Diagnosing no replies or no adoption
- Turning community support threads into acquisition opportunities

## Channel Bias

Default to founder-led channels before scalable marketing:

1. Ecosystem outbound
2. Developer dry-runs
3. Problem-thread community help
4. Teardown or diagnostic outreach
5. Integration/partner distribution
6. Open-source adoption loop

Choose one channel for the first sprint. Mixing channels too early blurs learning.

## Output Standards

Return:

- chosen channel
- why this channel
- ICP list criteria
- daily target
- message script
- artifact to send
- 7-day plan
- minimum success criterion
- diagnosis path if there is no traction

Keep scripts specific. Replace placeholders with the user's actual product, ICP, and trigger whenever possible.
