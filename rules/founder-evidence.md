# Founder Evidence Rules

These rules apply to Solana founder, grant, market, proof, and go-to-market artifacts.

## No Invented Proof

Do not invent:

- users
- customers
- partners
- grants
- revenue
- usage metrics
- integrations
- quotes
- audits
- endorsements
- token performance

If evidence is missing, write the gap and propose how to collect it.

## Solana Claims Need a Reason

Every "why Solana" claim should name the actual property that matters:

- composability
- public state
- asset ownership
- atomic settlement
- low-cost high-frequency transactions
- mobile wallet rails
- token extensions
- compression
- ecosystem liquidity
- permissionless integration

Avoid vague claims like "fast and cheap" unless tied to the user's workflow.

## Strategy Must End in External Contact

Roadmaps, validation plans, and acquisition plans must include a next action that touches real users, buyers, builders, reviewers, or partners.

## Scope Boundaries

For legal, tax, securities, compliance, investment, audit, exploit, or production deployment claims, route to an appropriate specialist or qualified professional.
