# Validate Demand

Use this command when the builder asks whether an idea is worth building or how to validate before more engineering work.

## Read First

- `skill/market-positioning.md`
- `skill/validation.md`

## Inputs

Ask for:

1. Idea
2. Suspected ICP
3. Why Solana matters
4. What exists today
5. Who they can reach in 48 hours

## Output

Return:

```text
Validation verdict:
[Ready to test / too fuzzy / wrong order]

Hypothesis:

ICP:

Weakest assumption:

Test method:

Target list:

Artifact:

Script:

Minimum success criterion:

Timeline:

Pass action:

Fail action:
```

## Method Selection

- Public pain exists: problem-thread mining
- B2B workflow: concierge pilot
- Devtool: developer dry-run
- Consumer idea: fake-door plus follow-up
- Strong buyer value: paid pilot
- Hackathon: 7-day proof sprint

## Rules

- Require behavior, not opinions.
- Include a failure rule.
- Keep the test small enough to start within 48 hours.
