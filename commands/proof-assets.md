# Proof Assets

Use this command when the builder provides raw evidence and needs case studies, grant updates, social proof, objection maps, or win/loss reports.

## Read First

- `skill/proof-assets.md`
- `skill/objection-map.md`
- `skill/win-loss.md`

## Choose Output

| Input | Output |
|-------|--------|
| One interview or pilot | Case study |
| Grant milestone notes | Milestone update |
| Public launch evidence | Launch proof post |
| Pitch or page risk | Objection map |
| Batch of notes | Win/loss report |

## Output Rules

- Cite claims when anchors exist.
- Label uncited claims.
- Keep quotes verbatim unless trimming with brackets.
- List gaps to close.
- End with the next ask.

## Default Output

```text
Evidence summary:

Best proof:

Weakest proof gap:

Asset:

Gaps to close:

Next ask:
```
