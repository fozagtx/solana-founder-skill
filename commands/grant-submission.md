# Grant Submission

Use this command for grant applications, hackathon submissions, bounty entries, demo scripts, and reviewer-ready project blurbs.

## Read First

- `skill/hackathon-grants.md`
- `skill/market-positioning.md`
- `skill/proof-assets.md`

## Inputs

Ask for:

1. Submission target
2. Project name and link
3. ICP
4. Problem
5. Demo status
6. Proof available
7. Requested grant/bounty/track
8. Deadline

## Output

Return:

```text
Readiness score:

One-line pitch:

Problem:

ICP:

Current workaround:

Solution:

Why Solana:

What works today:

Proof:

3-minute demo script:

Milestones:

Risks and mitigations:

Reviewer-ready blurb:

Submission checklist:

Next 72-hour action:
```

## Rules

- If proof is missing, design a proof sprint before polishing.
- If the demo is not working, scope the smallest working path.
- Never invent metrics, users, partners, or endorsement.
